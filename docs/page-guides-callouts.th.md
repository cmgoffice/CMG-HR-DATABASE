# ข้อความ Callout สำหรับภาพประกอบ

เอกสารนี้รวบรวมข้อความสั้นสำหรับวางบนภาพ annotated screenshot โดยจัดตามหน้าและตามภาพจริงที่ใช้ในคู่มือ

หากต้องส่งต่อให้ทีมออกแบบหรือคนทำสไลด์ ให้ใช้ไฟล์ `docs/page-guides-callouts-ppt-figma.th.md` ซึ่งจัด format เป็นบล็อกต่อภาพและหยิบไปวางใน PowerPoint/Figma ได้ง่ายกว่า

การจัดกลุ่มภาพในเอกสารนี้:
- `HR Dashboard` จำนวน 4 ภาพ
- `Project Dashboard` จำนวน 4 ภาพ
- `ประเมินผลพนักงาน` จำนวน 3 ภาพ

ภาพ `จัดการสิทธิ์การเข้าถึงตาม Role` ไม่รวมอยู่ในชุดนี้ตามขอบเขตงานปัจจุบัน

## HR Dashboard — ภาพ 1

มุมมองหลักของหน้า: แถบวันที่, KPI หลัก, โครงสร้างกำลังคน และแนวโน้มการมาทำงาน

1. ตำแหน่ง/ส่วนที่อธิบาย: `แถบวันที่และช่วงวิเคราะห์`  
   ข้อความสั้นสำหรับวางบนภาพ: ยืนยันช่วงวันที่ก่อนอ่านทุกค่า เพราะ KPI และกราฟทั้งหมดอิงช่วงนี้  
   หมายเหตุ: ถ้าเลือกช่วงวันที่ต่างกัน การเปรียบเทียบจะเปลี่ยนทันที

2. ตำแหน่ง/ส่วนที่อธิบาย: `KPI หลัก`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูสุขภาพแรงงานรวมของทั้งองค์กรในมุมคนมา ขาด ลา สาย OT และเคสเสี่ยง  
   หมายเหตุ: เป็นจุดเริ่มต้นสำหรับสแกนภาพรวมอย่างรวดเร็ว

3. ตำแหน่ง/ส่วนที่อธิบาย: `โครงสร้างกำลังคน`  
   ข้อความสั้นสำหรับวางบนภาพ: ดูว่ากำลังคนหลักกระจุกในกลุ่มใด เพื่อประเมินผลกระทบหากกลุ่มนั้นเริ่มมีปัญหา  
   หมายเหตุ: ช่วยชี้ว่าปัญหาของกลุ่มใดมีน้ำหนักต่อภาพรวมมากที่สุด

4. ตำแหน่ง/ส่วนที่อธิบาย: `แนวโน้มการมาทำงาน`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าปัญหาเกิดเฉพาะบางวัน หรือเกิดต่อเนื่องหลายวัน  
   หมายเหตุ: เหมาะสำหรับดู pattern ก่อนลงไปเจาะรายโครงการ

## HR Dashboard — ภาพ 2

มุมมองวิเคราะห์ความเสี่ยงและการกระจายพนักงานตามโครงการ

1. ตำแหน่ง/ส่วนที่อธิบาย: `ภาพรวมระดับความเสี่ยงพนักงาน`  
   ข้อความสั้นสำหรับวางบนภาพ: ดูว่าความเสี่ยงกระจุกอยู่ในระดับเฝ้าระวังหรือระดับสูง เพื่อจัดลำดับการติดตาม  
   หมายเหตุ: ใช้เป็นตัวช่วยจัดลำดับ ไม่ใช่ข้อสรุปสาเหตุสุดท้าย

2. ตำแหน่ง/ส่วนที่อธิบาย: `การกระจายพนักงานตามโครงการ`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าแรงงานส่วนใหญ่อยู่ในโครงการใด และโครงการไหนมีผลต่อภาพรวมมากที่สุด  
   หมายเหตุ: โครงการที่มีคนมากมักมีน้ำหนักต่อภาพรวมองค์กรสูง

3. ตำแหน่ง/ส่วนที่อธิบาย: `โครงการเสี่ยงจัดอันดับ`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้เรียงลำดับว่าโครงการไหนควรได้รับการติดตามก่อน  
   หมายเหตุ: เหมาะสำหรับใช้เลือกโครงการที่จะไปดูต่อใน `Project Dashboard`

4. ตำแหน่ง/ส่วนที่อธิบาย: `โครงการที่มีความเสี่ยง`  
   ข้อความสั้นสำหรับวางบนภาพ: อ่านสัดส่วนขาด ลา ค้าง และ OT ของแต่ละโครงการก่อนตัดสินใจเจาะลึก  
   หมายเหตุ: ใช้ประกอบการตัดสินใจว่าควรติดตามในมิติใดก่อน

## HR Dashboard — ภาพ 3

มุมมองคุณภาพข้อมูลและรายการพนักงานที่ต้องติดตาม

1. ตำแหน่ง/ส่วนที่อธิบาย: `ความครบถ้วนของข้อมูล`  
   ข้อความสั้นสำหรับวางบนภาพ: ตรวจคุณภาพข้อมูลก่อนตีความผล โดยเฉพาะกรณี attendance ยังไม่ครบ  
   หมายเหตุ: ถ้าข้อมูลไม่ครบ ควรระวังการสรุปผล KPI

2. ตำแหน่ง/ส่วนที่อธิบาย: `พนักงานที่ต้องติดตาม`  
   ข้อความสั้นสำหรับวางบนภาพ: นี่คือ action list หลักของหน้า ใช้จัดลำดับว่าจะติดตามใครก่อน  
   หมายเหตุ: ดูพร้อมกับ score ระดับความเสี่ยง และสาเหตุหลักของแต่ละคน

3. ตำแหน่ง/ส่วนที่อธิบาย: `คอลัมน์ล่าสุดและ Action`  
   ข้อความสั้นสำหรับวางบนภาพ: ดูว่าระบบพบเหตุล่าสุดเมื่อใด และควรประสานงานต่ออย่างไร  
   หมายเหตุ: เหมาะสำหรับใช้คุยต่อกับหัวหน้างานหรือ HR ประจำพื้นที่

## HR Dashboard — ภาพ 4

มุมมองพนักงานลา/ขาดของวันปัจจุบัน

1. ตำแหน่ง/ส่วนที่อธิบาย: `พนักงานลา/ขาดวันนี้`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ติดตามเหตุการณ์ของวันนี้โดยตรง แยกจากความเสี่ยงสะสมย้อนหลัง  
   หมายเหตุ: เหมาะกับการใช้ในมุม operation หน้างาน

2. ตำแหน่ง/ส่วนที่อธิบาย: `โครงการและตำแหน่ง/ประเภท`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าการขาดวันนี้กระทบโครงการใดและแรงงานกลุ่มไหน  
   หมายเหตุ: ช่วยประเมินผลกระทบต่อการทำงานของแต่ละโครงการ

3. ตำแหน่ง/ส่วนที่อธิบาย: `สถานะวันนี้`  
   ข้อความสั้นสำหรับวางบนภาพ: อ่านให้ชัดว่าเป็นลา หรือไม่มา เพื่อกำหนดวิธีติดตามต่อ  
   หมายเหตุ: วิธีติดตามหลังบ้านอาจต่างกันตามประเภทสถานะ

## Project Dashboard — ภาพ 1

มุมมองส่วนบนของหน้า: ตัวเลือกโครงการ, KPI หลัก และตารางสรุปกำลังคน

1. ตำแหน่ง/ส่วนที่อธิบาย: `ตัวเลือกโครงการ`  
   ข้อความสั้นสำหรับวางบนภาพ: ตรวจชื่อโครงการให้ถูกต้องก่อนอ่านตัวเลขทั้งหมด  
   หมายเหตุ: ทุกส่วนในหน้าอิงโครงการที่เลือก

2. ตำแหน่ง/ส่วนที่อธิบาย: `KPI หลักของโครงการ`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูภาพรวมทันทีว่าโครงการมีคนมาทำงานพอหรือเริ่มเสียสมดุล  
   หมายเหตุ: ครอบคลุมคนมา ขาด ลา ค้าง/ผิดโครงการ และ OT

3. ตำแหน่ง/ส่วนที่อธิบาย: `สรุปตามประเภทพนักงาน`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าปัญหาหนักสุดอยู่ที่แรงงานกลุ่มใดก่อนเจาะรายชื่อ  
   หมายเหตุ: เหมาะสำหรับหา root area ของปัญหา

4. ตำแหน่ง/ส่วนที่อธิบาย: `ตำแหน่งหลักในโครงการ`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าตำแหน่งใดเริ่มขาดสมดุลเมื่อเทียบกับจำนวนที่ควรมี  
   หมายเหตุ: เหมาะสำหรับติดตามผลกระทบต่อหน้างาน

## Project Dashboard — ภาพ 2

มุมมองแนวโน้ม coverage รายวันของโครงการ

1. ตำแหน่ง/ส่วนที่อธิบาย: `Coverage Trend`  
   ข้อความสั้นสำหรับวางบนภาพ: ดูว่า coverage ของโครงการดีขึ้นหรือแย่ลงในแต่ละวันเมื่อเทียบกับเป้าหมาย  
   หมายเหตุ: ช่วยหาช่วงเวลาที่ควรย้อนกลับไปวิเคราะห์สาเหตุ

2. ตำแหน่ง/ส่วนที่อธิบาย: `ค่าเปอร์เซ็นต์และ gap รายวัน`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้หาวันที่ coverage ตกมากที่สุด เพื่อเจาะดูสาเหตุของการขาดกำลังคน  
   หมายเหตุ: ควรใช้ร่วมกับข้อมูลลา/ขาดและกลุ่มแรงงานที่เกี่ยวข้อง

## Project Dashboard — ภาพ 3

มุมมองวิเคราะห์ coverage ของโครงการ

1. ตำแหน่ง/ส่วนที่อธิบาย: `Coverage Analysis`  
   ข้อความสั้นสำหรับวางบนภาพ: ดู coverage รวมของโครงการก่อนว่าคนมาจริงเพียงพอหรือยังมี gap  
   หมายเหตุ: เป็นภาพรวมระดับโครงการก่อนแยกดูรายกลุ่ม

2. ตำแหน่ง/ส่วนที่อธิบาย: `Coverage ตามประเภทพนักงาน`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่ากลุ่มแรงงานใดเริ่มขาด coverage หรือพึ่ง OT สูงผิดปกติ  
   หมายเหตุ: ช่วยแยกว่าปัญหาอยู่ที่ประเภทแรงงานใด

3. ตำแหน่ง/ส่วนที่อธิบาย: `Critical Role Coverage`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าตำแหน่งสำคัญใดต่ำกว่าแผนหรือ assigned distribution  
   หมายเหตุ: เหมาะสำหรับติดตามตำแหน่งที่กระทบหน้างานโดยตรง

## Project Dashboard — ภาพ 4

มุมมองรายการติดตามและพนักงานลา/ขาดวันนี้ของโครงการ

1. ตำแหน่ง/ส่วนที่อธิบาย: `รายการที่ต้องติดตาม`  
   ข้อความสั้นสำหรับวางบนภาพ: เป็นรายการ follow-up ย้อนหลังของโครงการ ให้เริ่มจากคนที่มีสัญญาณซ้ำหรือสะสม  
   หมายเหตุ: ใช้สำหรับวางแผนติดตามเชิงรุก

2. ตำแหน่ง/ส่วนที่อธิบาย: `พนักงานลา/ขาดวันนี้`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ติดตามผลกระทบหน้างานของวันนี้โดยตรง  
   หมายเหตุ: แยกจากรายการความเสี่ยงย้อนหลังอย่างชัดเจน

3. ตำแหน่ง/ส่วนที่อธิบาย: `สถานะด้านขวา`  
   ข้อความสั้นสำหรับวางบนภาพ: อ่านให้ชัดว่าเป็นลา หรือไม่มา เพื่อกำหนดการประสานงานต่อ  
   หมายเหตุ: ช่วยให้ทีมปฏิบัติการตัดสินใจได้เร็วขึ้น

## ประเมินผลพนักงาน — ภาพ 1

มุมมองแท็บ `ประเมิน`: หัวข้อหน้า, สิทธิ์ของผู้ใช้, ตัวกรอง และรายการชุดในรอบ

1. ตำแหน่ง/ส่วนที่อธิบาย: `หัวข้อหน้าและสิทธิ์ของคุณ`  
   ข้อความสั้นสำหรับวางบนภาพ: ดูสิทธิ์ของตนเองก่อน เพราะหน้าที่ที่ทำได้ขึ้นกับ Tier และการมอบหมาย  
   หมายเหตุ: ผู้ใช้คนละสิทธิ์จะเห็นงานที่ทำได้ต่างกัน

2. ตำแหน่ง/ส่วนที่อธิบาย: `แท็บ ประเมิน / มอบหมายชุด / สรุปผล`  
   ข้อความสั้นสำหรับวางบนภาพ: หน้าเดียวรองรับทั้งการทำงานตามรอบ การมอบหมายผู้ประเมิน และการดูผลสรุป  
   หมายเหตุ: ให้เลือกแท็บตามบทบาทที่กำลังทำ

3. ตำแหน่ง/ส่วนที่อธิบาย: `ตัวกรองรอบ โครงการ และรายการชุด`  
   ข้อความสั้นสำหรับวางบนภาพ: เลือกรอบให้ถูกต้อง แล้วดูว่าชุดใดถึงคิวคุณและอยู่ที่ Tier ใด  
   หมายเหตุ: ปุ่มท้ายแถวจะบอกทันทีว่าต้องให้คะแนน ตรวจ/อนุมัติ หรือดูอย่างเดียว

## ประเมินผลพนักงาน — ภาพ 2

มุมมองแท็บ `มอบหมายชุด` สำหรับกำหนดผู้ประเมินและผู้ตรวจของแต่ละชุด

1. ตำแหน่ง/ส่วนที่อธิบาย: `โครงการและชุดแรงงาน`  
   ข้อความสั้นสำหรับวางบนภาพ: หนึ่งชุดคือหนึ่งหน่วยงานประเมิน และหนึ่งเส้นทางการอนุมัติ  
   หมายเหตุ: การมอบหมายผูกกับคู่โครงการและชุด ไม่ได้ผูกทั้งโครงการรวมกัน

2. ตำแหน่ง/ส่วนที่อธิบาย: `Tier 1`  
   ข้อความสั้นสำหรับวางบนภาพ: ต้องมีผู้ประเมินอย่างน้อย 2 คน เพื่อให้รอบเดินต่อไป Tier 2 ได้  
   หมายเหตุ: หากมอบหมายไม่ครบ ระบบจะไม่เปิดการทบทวนระดับถัดไป

3. ตำแหน่ง/ส่วนที่อธิบาย: `Tier 2`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้กำหนดผู้ตรวจระดับชุด ที่จะทบทวนหรือ override รายคนเมื่อจำเป็น  
   หมายเหตุ: ควรเลือกผู้ที่มีหน้าที่ตัดสินใจในระดับชุดได้จริง

## ประเมินผลพนักงาน — ภาพ 3

มุมมองแท็บ `สรุปผล` สำหรับอ่านผลประเมินและส่งออกข้อมูล

1. ตำแหน่ง/ส่วนที่อธิบาย: `ตัวกรองและปุ่มส่งออก CSV`  
   ข้อความสั้นสำหรับวางบนภาพ: เลือกรอบ โครงการ ประเภท และชุดก่อนอ่านผล หรือส่งออกข้อมูลต่อ  
   หมายเหตุ: เหมาะสำหรับเตรียมรายงานรายรอบหรือสรุปผู้ที่ต้องติดตาม

2. ตำแหน่ง/ส่วนที่อธิบาย: `สรุปตามชุดแรงงาน`  
   ข้อความสั้นสำหรับวางบนภาพ: ใช้ดูว่าชุดใดควรได้รับความสนใจก่อนจากค่าเฉลี่ยและจำนวนคนเฝ้าระวัง  
   หมายเหตุ: เป็นจุดเริ่มต้นก่อนเปิดดูผลรายบุคคล

3. ตำแหน่ง/ส่วนที่อธิบาย: `คะแนน ผลลัพธ์ และที่มาของคะแนน`  
   ข้อความสั้นสำหรับวางบนภาพ: ดูทั้งคะแนนสุดท้ายและที่มาว่ามาจากเฉลี่ย Tier 1 หรือ override ของ Tier ที่สูงกว่า  
   หมายเหตุ: ควรอ่านคู่กับสถานะรอบเพื่อแยกผลชั่วคราวกับผลปิดรอบ
