# แก้ไขข้อมูล Project No. ที่ผิดรูปแบบ

## ปัญหา
ในฐานข้อมูลพนักงาน ฟิลด์ "สถานะโครงการ" บางรายการบันทึกเป็นชื่อโครงการแทนรหัสโครงการ เช่น:
- ❌ `PRJ-2026-J-Workshop` (ผิด - ใช้ชื่อโครงการ)
- ✅ `PRJ-2026-J-002` (ถูก - ใช้รหัสโครงการ)

ทำให้การแสดงผลแสดงเป็น `J-Workshop` แทนที่จะเป็น `J-002`

## วิธีแก้ไข

### ขั้นตอนที่ 1: ตรวจสอบข้อมูล (ไม่แก้ไข)
```bash
node src/scripts/fixWorkshopProjectNo.js
```

สคริปต์จะแสดง:
- จำนวนพนักงานที่ต้องแก้ไข
- รายการที่จะเปลี่ยนแปลง (เก่า → ใหม่)
- **ไม่มีการแก้ไขข้อมูลจริง**

### ขั้นตอนที่ 2: แก้ไขข้อมูล (ต้องยืนยัน)
```bash
node src/scripts/fixWorkshopProjectNo.js --confirm
```

สคริปต์จะ:
1. โหลดข้อมูลโครงการทั้งหมด
2. สร้าง mapping ระหว่างชื่อโครงการกับรหัสโครงการ
3. ตรวจสอบพนักงานทุกคน
4. แก้ไขข้อมูลที่ผิดรูปแบบ
5. แสดงผลสรุป

## หมายเหตุ

- ✅ ปลอดภัย: ตรวจสอบก่อนแก้ไขจริง
- ✅ รองรับทั้งโครงการเดียวและหลายโครงการ
- ✅ แสดงรายละเอียดการเปลี่ยนแปลง
- ⚠️  ต้องมีไฟล์ `.env` ที่ถูกต้อง
- ⚠️  ควร backup ข้อมูลก่อนรัน `--confirm`
